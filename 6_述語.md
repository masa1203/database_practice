### LIKE述語

文字列の部分一致検索を行うときにつかう。
dddを含むレコードを選択することを考える。

- 前方一致
    - dddが最初に位置しているレコードだけが選択される検索のしかた
- 中間一致
    - dddが検索対象の文字列のどこかに含まれていれば選択される
- 後方一致
    - dddが文字列の最後尾にあるレコードだけが選択対象になる

- 「%」は **0以上の任意の文字列** を意味する。

``` SQL
-- DDL: テーブル作成
create table SampleLike
(
    strcol varchar(6) not null,
    primary key (strcol)
);

-- DML: データ登録
start transaction;
insert into SampleLike (strcol) values ('abcddd');
insert into SampleLike (strcol) values ('dddabc');
insert into SampleLike (strcol) values ('abdddc');
insert into SampleLike (strcol) values ('abcdd');
insert into SampleLike (strcol) values ('ddabc');
insert into SampleLike (strcol) values ('abddc');
commit;

-- 確認
mysql> select * from SampleLike;
+--------+
| strcol |
+--------+
| abcdd  |
| abcddd |
| abddc  |
| abdddc |
| ddabc  |
| dddabc |
+--------+
6 rows in set (0.00 sec)

-- 前方一致検索
select *
from SampleLike
where strcol LIKE 'ddd%';
+--------+
| strcol |
+--------+
| dddabc |
+--------+
1 row in set (0.00 sec)

-- 中間一致検索
select *
from SampleLike
where strcol LIKE '%ddd%';
+--------+
| strcol |
+--------+
| abcddd |
| abdddc |
| dddabc |
+--------+
3 rows in set (0.00 sec)

-- 後方一致検索
select *
from SampleLike
where strcol LIKE '%ddd';
+--------+
| strcol |
+--------+
| abcddd |
+--------+
1 row in set (0.00 sec)

-- abc + 任意の二文字
select *
from SampleLike
where strcol LIKE 'abc__';
+--------+
| strcol |
+--------+
| abcdd  |
+--------+
1 row in set (0.01 sec)

-- abc + 任意の3文字
-- 4文字はEmpty
select *
from SampleLike
where strcol LIKE 'abc____';
+--------+
| strcol |
+--------+
| abcddd |
+--------+
1 row in set (0.00 sec)
```

## BETWEEN述語(範囲検索)

- 引数を3つもつ
- between は両端も含む

``` SQL

from Shohin

3 rows in set (0.00 sec)
```

## IS NULL, IS NOT NULL(NULLか非NULL判定)

- ある列がNULLの行を選択するとき、=では判定できない。
    - **IS NULL** 述語を使って判定する

``` SQL
select shohin_mei, shiire_tanka
from Shohin
where shiire_tanka is NULL;
+-----------------+--------------+

+-----------------+--------------+
| フォーク        |         NULL |
| ボールペン      |         NULL |
+-----------------+--------------+

select shohin_mei, shiire_tanka

+-----------------------+--------------+

```

## IN述語

where shiire_tanka = 320,
shiire_tanka = 500,
...

とやって条件分岐するのは大変。INを使う。該当なしはEmpty。

- **NULLは選択できない**

```SQL
-- IN
select shohin_mei, shiire_tanka
from Shohin
where shiire_tanka in (320, 500, 5000);
+--------------------+--------------+
| shohin_mei         | shiire_tanka |
+--------------------+--------------+
| Tシャツ            |          500 |
| 穴あけパンチ       |          320 |
| 圧力なべ           |         5000 |
+--------------------+--------------+
3 rows in set (0.00 sec)

-- NOT IN
select shohin_mei, shiire_tanka
from Shohin
where shiire_tanka not in (320, 500, 5000);
+-----------------------+--------------+
| shohin_mei            | shiire_tanka |
+-----------------------+--------------+
| カッターシャツ        |         2800 |
| 包丁                  |         2800 |
| おろし金              |          790 |
+-----------------------+--------------+
3 rows in set (0.00 sec)
```

## EXISTS述語

- INとNOT IN　でほぼ代用できる

- **ある条件に合致するレコードの存在有無を調べること**
    - 存在すればTRUE、存在しなければFALSEを返す

- 引数を1つしかとらない述語(右側に引数を1つだけかく)
    - 引数は常に相関サブクエリ

- 「select * 」はSQLの一種の慣習

``` SQL
-- INでかかれたのものをEXISTSで書き換える
select shohin_mei, hanbai_tanka
from Shohin AS S
where EXISTS (select *
                from TenpoShohin as TS
                where TS.tenpo_id = '000C'
                and TS.shohin_id = S.shohin_id);
+-----------------------+--------------+
| shohin_mei            | hanbai_tanka |
+-----------------------+--------------+
| カッターシャツ        |         4000 |
| 包丁                  |         3000 |
| フォーク              |          500 |
| おろし金              |          880 |
+-----------------------+--------------+
4 rows in set (0.03 sec)

-- 東京店に置いてある商品以外
-- 存在しない場合にTrueを返す
select shohin_mei, hanbai_tanka
from Shohin AS S
where NOT EXISTS (select *
                    from TenpoShohin as TS
                    where tenpo_id = '000A'
                    and TS.shohin_id = S.shohin_id);
```
